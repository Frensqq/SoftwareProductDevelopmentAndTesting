# Определитель вида треугольника

> Данный проект представляет собой Android-приложение, которое по трём сторонам определяет тип треугольника.

![Треугольник](https://media.tenor.com/MMhvr_YdWTgAAAAi/yellow-triangle.gif)

---

# Описание

Приложение позволяет пользователю ввести длины трёх сторон и получить результат:

- **разносторонний**
- **равнобедренный**
- **равносторонний**

При некорректном вводе (буквы, отрицательные числа, нарушение неравенства треугольника) выводится диалог с ошибкой.

---

## Возможности

+ Проверка существования треугольника
+ Определение типа треугольника
+ ~~Обработка дробных чисел~~
+ Обработка ошибок ввода
+ Splash screen загрузки

---


## Как работает данное приложение

1. Пользователь вводит три числа.
2. Приложение проверяет:
   - Все ли поля заполнены?
   - Являются ли введённые значения целыми числами?
   - Больше ли каждое число нуля?
   - Выполняется ли неравенство треугольника?
> Сумма любых двух сторон должна быть больше третьей.
3. Если все проверки пройдены:
   - Сравниваются стороны.
   - Выводится тип треугольника.
4. Если ошибка:
   - Показывается диалог с описанием проблемы.

# Структура проекта

## 1. Логика (`Logic/TypeDefinition.kt`)

Файл содержит функцию `defineTriangle` и sealed-класс `TriangleResult`.

## Sealed-класс TriangleResult

`TriangleResult` — это sealed-класс, который представляет возможные результаты работы функции `defineTriangle`. Он находится в файле `Logic/TypeDefinition.kt`.

Использование sealed-класса позволяет одной функции возвращать как Int при успешном запросе так и ошибку с лучае её появлении 

```kotlin
sealed class TriangleResult {
    data class Success(val type: Int) : TriangleResult()
    data class Error(val title: String, val description: String) : TriangleResult()
}
```
**Варианты результата**

|Вариант|	Описание|	Поля|
|--------|------------|-------|
|`Success`|	Треугольник существует и его тип определён|	type: Int — код типа (0, 1 или 2)|
|`Error`|	Произошла ошибка валидации|	title: String — заголовок ошибки, description: String — пояснение|

Коды типа треугольника:\
0 — разносторонний (все стороны разные)\
1 — равнобедренный (две стороны равны)\
2 — равносторонний (все три стороны равны)

## Функция defineTriangle

Функция `defineTriangle` находится в файле `Logic/TypeDefinition.kt`. Она принимает три строковых параметра и возвращает `TriangleResult`.

**Параметры:**  

|Имя |     Тип      | Описание|
|--  | -------------| ------------------------------|
|`a`| `String` | Сторона треугольника a|
|`b`| `String` | Сторона треугольника b|
|`c`| `String` | Сторона треугольника c|

**Возвращаемое значение:**  

|     Тип      | Описание|
| -------------| ------------------------------|
| `TriangleResult` | Тип кодировки из перечисления|

**Алгоритм работы**\

1) Преобразование в числа

   - Попытка конвертировать каждую строку в Int

   - При ошибке → возврат Error("Ошибка ввода", "Сторона X должна быть целым числом")

2) Проверка положительных значений

    - Если `aD <= 0 || bD <= 0 || cD <= 0 `
    - возврат Error("Некорректные значения", "Все стороны должны быть больше 0")

3) Проверка неравенства треугольника

    - Сумма любых двух сторон должна быть больше третьей
    - Возврат Error("Треугольник не существует", "Сумма любых двух сторон должна быть больш третьей")

Условие: 
```kotlin
if (aD + bD <= cD || aD + cD <= bD || bD + cD <= aD)
```
  
4) Проверка неравенства треугольника

```kotlin
val type = when {
    aD == bD && bD == cD -> 2  // равносторонний
    aD == bD || bD == cD || aD == cD -> 1  // равнобедренный
    else -> 0  // разносторонний
}
```

5) Возврат успешного результата

    - TriangleResult.Success(type)

**Пример и вызов результатов**

|     Вызов      | Результат|
| -------------| ------------------------------|
| defineTriangle("3", "3", "3") | Success(2) — равносторонний|
| defineTriangle("0", "5", "5") | Error("Некорректные значения", "Все стороны должны быть больше 0")|

**Код функции:**
```kotlin
fun defineTriangle(a: String, b: String, c: String): TriangleResult {

    val aD: Int
    val bD: Int
    val cD: Int

    try {
        aD = a.toInt()
    } catch (e: Exception) {
        return TriangleResult.Error(
            "Ошибка ввода",
            "Сторона A должна быть целым числом"
        )
    }

    try {
        bD = b.toInt()
    } catch (e: Exception) {
        return TriangleResult.Error(
            "Ошибка ввода",
            "Сторона B должна быть целым числом"
        )
    }

    try {
        cD = c.toInt()
    } catch (e: Exception) {
        return TriangleResult.Error(
            "Ошибка ввода",
            "Сторона C должна быть целым числом"
        )
    }
    if (aD <= 0 || bD <= 0 || cD <= 0) {
        return TriangleResult.Error(
            "Некорректные значения",
            "Все стороны должны быть больше 0"
        )
    }

    if (aD + bD <= cD || aD + cD <= bD || bD + cD <= aD) {
        return TriangleResult.Error(
            "Треугольник не существует",
            "Сумма любых двух сторон должна быть больше третьей"
        )
    }

    val type = when {
        aD == bD && bD == cD -> 2
        aD == bD || bD == cD || aD == cD -> 1
        else -> 0
    }

    return TriangleResult.Success(type)
}
```

## Navigation

### NavigationRoutes.kt

- Позволяет осществить переход от SplashScren на ProgramScreen 

**Маршркты навигации:**
```kotlin
object NavigationRoutes {
    const val SPLASH = "splash"
    const val MAIN = "main"
}
```

|Элемент|	Описание|
|----------|------------|
|rememberNavController()	|Создаёт и сохраняет контроллер навигации|
|NavHost	|Контейнер для навигации|
|startDestination	|Стартовый экран (SPLASH)|
|composable	|Регистрирует экран с указанным маршрутом|

**Функция Навигации:**
```kotlin
@Composable
fun Navigation() {
    val navController = rememberNavController()
    NavHost(navController = navController, startDestination = NavigationRoutes.SPLASH) {

        composable(NavigationRoutes.SPLASH){
            SplashScreenCustom(navController)
        }
        composable(NavigationRoutes.MAIN){
            ProgramScreen()
        }

    }
}
```
